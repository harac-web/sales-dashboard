## データが全部0になる

### パターン1: monthSortのスケール不一致

**症状**: あるタブのデータが全月0。他のタブは正常。

**原因**: データソースによって monthSort のスケールが異なる場合に発生する。

| データソース | 月フォーマット | monthSort |
|------------|-------------|----------|
| ISO日付（Googleスプレッドシートの日付セル）| `"2025-07-01T..."` | 202507 |
| テキスト `"2025年7月"` | `"2025年7月"` | 202507 |
| テキスト `"25年7月"` | `"2025年7月"` | 202507 |
| テキスト `"25.7月"` | `"2025年7月"` | 202507 |

すべて202507になるのが正常。`parseMonth()` が壊れているか、スプレッドシートの月フォーマットが想定外の形式になっているとズレが起きる。

**デバッグ手順**:
1. ブラウザのコンソール（F12）を開く
2. `seikyuuData.slice(0,2)` 等をコンソールで直接確認
3. `monthSort` が6桁（202507）か確認。4桁（2507）や2桁（25）になっていれば異常

**よくある原因**:
- `parseMonth()` のISO日付ブランチで `getUTCFullYear() % 100` になっていると2桁年になる → `getUTCFullYear()` に修正
- `mTo` の上限値が `99999` だと monthSort=202507 のレコードが除外される → `999999` に修正

### パターン2: カテゴリ名の不一致

**症状**: 特定ページのみ全部0。他ページは正常。

**原因**: フロント側のハードコード定数とスプレッドシートのカテゴリ名が一致していない。

**チェックポイント**:

| 確認項目 | NG例 | OK例 |
|---------|------|------|
| 全角/半角括弧 | `広告運用費(立替費のみ)` | `広告運用費（立替費のみ）` |
| カナの濁点 | `プリパ印刷` | `プリバ印刷` |
| スペースの有無 | `人件費 （社員）` | `人件費（社員）` |

**デバッグ手順**:
```javascript
// コンソールで実行（立替費の場合）
[...new Set(seikyuuData.map(r => r.cat))]
// → 実際のカテゴリ名一覧が出る
```
フロント側の定数と1文字ずつ比較する。

**フロント側のハードコード定数の場所**:
| 定数 | 場所 | 値 |
|------|------|-----|
| `CAT_KOUKOKU` | `renderSeikyuuPage()` | `'広告運用費（立替費のみ）'` |
| `CAT_PURIPA` | `renderSeikyuuPage()` | `'プリバ印刷・交通費等（内税）'` |
| 人件費フィルター | `renderRiekiPage()` | `'人件費'` / `'人件費（社員）'` |

### パターン3: GASが該当フィールドを返していない

**症状**: `seikyuuData.length = 0` または `costData.length = 0`。

**原因**: GASのレスポンスに期待するフィールドが含まれていない。

**デバッグ手順**:
1. コンソールで `seikyuuData.length` を確認
2. 0なら GAS 側の問題。GAS エディタのログを確認する
3. GAS の `doGet()` が `seikyuuData` フィールドを返しているか確認

## 特定のカテゴリだけ0になる

**原因候補**:
1. そのカテゴリのレコードだけ月フォーマットが異なる（シートの一部行だけ日付セル等）
2. 金額が円単位と万円単位で混在している

**金額単位の確認**:
| シート | 金額の単位 | フロントでの処理 |
|-------|----------|--------------|
| 通常コスト参照 | 円 | ÷ 10000 |
| 社員人件費 | 万円 | そのまま（`sheetSource='employee'`で判定） |
| 請求金額参照 | 円 | ÷ 10000 |

社員人件費の `sheetSource='employee'` がGASで付与されていない場合、金額が1/10000になる。

## 月の列が表示されない / 列が増えすぎる

**症状A**: 特定のタブで月列が0列。

**原因**: `tatekaeFiltered` / `kyoekiFiltered` / `costFiltered` のすべてが空。期フィルターか月ソートの問題。

**症状B**: 月列が重複する（`"2025年7月"` と `"25年7月"` が別列として出る）。

**原因**: 同じ月が異なる `monthLabel` 形式で存在している。`parseMonth()` が統一できていない。

**デバッグ手順**:
```javascript
// コンソールで実行
[...new Set(costData.map(r => r.monthLabel))].sort()
[...new Set(seikyuuData.map(r => r.monthLabel))].sort()
// → 両方が同じ形式（"2025年7月"）であること
```

## 期フィルターが効かない / 全期間が混在する

**原因**: `kiMonths` は `rawData`（売上データ）から期→月範囲を生成する。コストデータが別の期定義を持っていても無視される。

**仕様**: 期の月範囲は rawData の `ki` フィールドから自動生成。rawData に該当期のデータがない場合、その期はフィルターに表示されない。

## Divフィルターの選択肢がおかしい

**症状**: 選択肢に本来出るべきでないカテゴリが出る（例：以前は「社員人件費」が選択肢に出ていた）。

**原因**: `normCost()` で `Div名` が正しく設定されていない。

**チェック**:
```javascript
// コンソールで実行
[...new Set(costData.map(r => r['Div名']))].sort()
// → "社員人件費" 等の異常な値が含まれていないか確認
```

**社員人件費の `Div名` 正規化仕様**:
```javascript
if (r.sheetSource === 'employee') {
  normalized['大分類'] = '人件費（社員）';
  normalized['Div名'] = r['Div'] || '';  // シートの "Div" 列を使う
}
```
GASが `Div` フィールドを返していない場合、`Div名` が空になりフィルターに影響する。

## 3階層展開が機能しない

**症状**: 人件費（社員）の行をクリックしても種別（フロント/プロダクト）が展開されない。

**確認チェックリスト**:
1. `r['種別']` にデータがあるか → コンソールで `costData.filter(r => r['大分類'] === '人件費（社員）').map(r => r['種別'])`
2. deepView モード（Divフィルター選択時）か通常モードかで異なるコードパスを通る
3. deepView時: `isEmpCat` フラグが正しく `cat === '人件費（社員）'` を検出しているか

## ページ全体が真っ白 / ロードエラー

**原因候補**:

| 原因 | 確認方法 |
|------|---------|
| GAS APIがエラーを返している | コンソールの Network タブで API レスポンスを確認 |
| GASのデプロイが古い | GASエディタで「デプロイを管理」→最新版か確認 |
| APIキーが違う | `index.html` の `API_KEY` 定数を確認（`"yomi-shukei-test"`） |
| 認証エラー | コンソールに `checkPermission` 関連のエラーがないか確認 |

**GAS APIレスポンスの直接確認**:
ブラウザで以下URLにアクセス（要Googleログイン）:
```
https://script.google.com/macros/s/<DEPLOY_ID>/exec?key=yomi-shukei-test
```
`{"success":true,...}` が返れば正常。`{"success":false,"error":"..."}` ならエラーメッセージを確認。

## デバッグ用コンソールコマンド集

```javascript
// データ件数確認
rawData.length
costData.length
seikyuuData.length
corpoCostData.length

// カテゴリ一覧
[...new Set(costData.map(r => r['大分類']))].sort()
[...new Set(seikyuuData.map(r => r.cat))].sort()

// 月ラベル一覧（フォーマット確認）
[...new Set(costData.map(r => r.monthLabel))].sort()
[...new Set(seikyuuData.map(r => r.monthLabel))].sort()

// monthSortの範囲確認
Math.min(...seikyuuData.map(r => r.monthSort))
Math.max(...seikyuuData.map(r => r.monthSort))

// 社員人件費の確認
costData.filter(r => r.sheetSource === 'employee').slice(0, 3)

// 特定カテゴリの金額確認
costData.filter(r => r['大分類'] === '人件費（社員）').map(r => ({month: r.monthLabel, amt: r.amt}))
```
