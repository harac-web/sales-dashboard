# ヨミクエリ 売上ダッシュボード 仕様書

最終更新: 2026-05-08

---

## 1. システム構成

```
Googleスプレッドシート
    ↓ Google Apps Script (GAS)
REST API (GAS Web App)
    ↓ fetch()
index.html (静的シングルページ)
    ↓ GitHub Pages
https://harac-web.github.io/sales-dashboard/
```

### 認証フロー
1. Google OAuth でメールアドレスを取得
2. GAS の `checkPermission(email, token)` を呼び出し
3. レスポンスの `tabs` 配列で表示可能なページを制御

## 3. スプレッドシート側の仕様

### 3-1. シート一覧

| シート名 | GASでの変数名 | 用途 |
|---------|-------------|------|
| 通常コスト参照 | costRows | 部門コスト（インターン・外注等） |
| 社員人件費 | costRows（sheetSource='employee'） | 社員給与コスト |
| コーポレートコスト | corpoCostData | 全社共通コスト |
| 請求金額参照 | seikyuuData | 立替費（横長→縦長にピボット） |
| 売上データシート | rawData | 売上明細 |
| 取引先マスタ | companyMap | 取引先の属性情報 |

### 3-2. 各シートのカラム定義

#### 通常コスト参照（部門コスト）
| カラム名 | 型 | 説明 |
|---------|-----|------|
| 年月 | テキストまたは日付 | 例: `"2025年7月"` または ISO日付 |
| 金額（税別） | 数値 | **万円単位**（例: `150` = 150万円） |
| 大分類 | テキスト | カテゴリ名（例: `人件費`, `機械費`） |
| Div名 | テキスト | 部門名 |
| TM | テキスト | チーム名 |
| 支払い区分 | テキスト | 支払い種別 |

#### 社員人件費
| カラム名 | 型 | 説明 |
|---------|-----|------|
| 年月 | テキスト | 例: `"2025年7月"` |
| TM | テキスト | チーム名 |
| Div | テキスト | 部門名（GASで `Div名` にマッピング） |
| 種別 | テキスト | `フロント` / `プロダクト` 等 |
| 金額（税別） | 数値 | **万円単位** |

> **注意**: 通常コストの金額（税別）は円単位のためGASで÷10000するが、社員人件費は万円単位のためそのまま使用。GASが `sheetSource='employee'` を付与することでフロント側で区別する。

#### 請求金額参照（立替費）
横長形式: カテゴリ行 × 月列

| カラム | 説明 |
|-------|------|
| 1列目 | カテゴリ名（例: `広告運用費（立替費のみ）`） |
| 2列目以降 | 各月の金額（円単位）。ヘッダーは `"25.7月"` 等のテキスト形式 |

GASがこれを縦長（1レコード＝1カテゴリ×1月）に変換して返す。

**使用するカテゴリ名（フロント側のハードコード値と一致が必要）:**
- `広告運用費（立替費のみ）` — 外税対象
- `プリバ印刷・交通費等（内税）` — 内税対象

> **注意**: 全角括弧（`（）`）と半角括弧（`()`）は別文字として扱われる。カテゴリ名が完全一致しないとデータが0になる。

## 4. GAS API 仕様

### エンドポイント
```
GET https://script.google.com/macros/s/<DEPLOY_ID>/exec?key=yomi-shukei-test
```

### レスポンス形式
```json
{
  "success": true,
  "updated": "2025-07-01T12:00:00Z",
  "count": 1234,
  "data": [...],
  "costData": [...],
  "corpoCostData": [...],

  "companies": [...]

```

### 各データのフィールド

#### `data`（売上明細）
| フィールド | 説明 |
|-----------|------|
| 計上月 | ISO日付または日本語形式 |
| 売上(万円) | 数値（既に万円単位） |
| ヨミコード | CD |
| 担当 | 担当者名 |
| 取引先 | 会社名 |
| 商品名 | 商品名 |
| ステータス | `J` / `B` 等 |
| 大分類 | カテゴリ |
| 製品タイプ | タイプ |
| 売上TM | チーム |
| 売上Div | 部門 |
| 期 | `4期` / `5期` 等 |

#### `costData` / `corpoCostData`
| フィールド | 説明 |
|-----------|------|
| 年月 | ISO日付または `"2025年7月"` 形式 |
| 金額（税別） | 円単位（÷10000で万円化）|
| 大分類 | カテゴリ（`人件費`, `機械費` 等） |
| Div名 | 部門名 |
| TM | チーム名 |
| 支払い区分 | 支払い種別 |
| sheetSource | `'employee'` の場合は社員人件費 |
| Div | 社員人件費の場合の部門名（`Div名` の元） |

#### `seikyuuData`
| フィールド | 説明 |
|-----------|------|
| month | `"25年7月"` 形式（GASがピボット時に変換） |
| amt | 円単位の金額（÷10000で万円化） |
| cat | カテゴリ名（シートの行ラベル） |
| rowIndex | 元シートの行番号 |

## 5. フロントエンド仕様

### 5-1. データ正規化

#### parseMonth(v)
月文字列を統一フォーマットに変換する。

| 入力例 | 出力 label | 出力 sort |
|-------|-----------|----------|
| `"2025-07-01T..."` | `"2025年7月"` | `202507` |
| `"2025年7月"` | `"2025年7月"` | `202507` |
| `"25年7月"` | `"2025年7月"` | `202507` |
| `"25.7月"` | `"2025年7月"` | `202507` |

> **規約**: `sort` の値は `YYYYMM` の6桁整数（例: 202507）。フィルターの上限値として `999999` を使用（`99999` では6桁のsortが除外されてしまう）。

#### normCost(r) — costData / corpoCostData 正規化
```
入力: GASから返ってきた生のコストレコード
出力: {
  ...元フィールド,
  monthLabel: "2025年7月",    // parseMonth(r['年月']) のlabel
  monthSort: 202507,          // parseMonth(r['年月']) のsort
  amt: <万円>,                // sheetSource='employee' なら金額そのまま、他は÷10000
  大分類: "人件費（社員）",    // sheetSource='employee' のみ上書き
  Div名: r['Div'] || ''       // sheetSource='employee' のみ上書き
}
```

#### seikyuuData 正規化
```
入力: GASから返ってきた立替費レコード
出力: {
  ...元フィールド,
  monthLabel: "2025年7月",
  monthSort: 202507,
  amt: r.amt / 10000,         // 円→万円
  rowIndex: <数値>
}
```

### 5-3. ページ別の大分類ハードコード値

#### 部門コスト（renderCostPage）

## 6. 既知の注意点・落とし穴

### 月フォーマットの統一
- **全データの monthSort は YYYYMM の6桁整数であること**
- `mTo` の上限は `999999`（以前は `99999` で202507が除外されるバグがあった）
- ISO日付の場合、`getUTCFullYear()` をそのまま使う（`% 100` で2桁化しない）

### 社員人件費の金額単位
- 社員人件費シートの金額は**万円単位**
- 通常コストシートの金額は**円単位**（GASが÷10000）
- GASは `sheetSource='employee'` を付与して区別し、フロントでも `sheetSource='employee'` の場合のみ÷10000しない

### 大分類・カテゴリ名の完全一致
- フロントとスプレッドシートのカテゴリ名は**文字列完全一致**
- 全角/半角括弧（`（）` vs `()`）、スペースの有無で別扱い
- `プリバ` と `プリパ` も別文字

### Divフィルター選択時の表示モード
- `F.costDiv` や `F.costTm` が設定されると `deepView` モードに切り替わり、テーブルの階層構造が変わる

## 7. 今後の変更時のチェックリスト

スプレッドシートのカテゴリ名を変更する場合:
- [ ] フロント側のハードコード定数（CAT_KOUKOKU, CAT_PURIPA, jinkenFiltered の大分類文字列等）を更新

新しいシートを追加する場合:
- [ ] GASに読み込み処理を追加
- [ ] GASのレスポンスに新しいフィールドを追加
- [ ] フロントの `fetchData()` に正規化処理を追加
- [ ] `parseMonth()` で月フォーマットが扱えるか確認

月フォーマットを変更する場合:
- [ ] GASが返す `month` フィールドが `parseMonth()` で処理できる形式か確認
- [ ] monthSort が YYYYMM の6桁であることを確認
